# PCS Modules

This document describes the responsibilities of major PCS repository modules. It does not define implementation details.

## PCS_VARIABLE_REGISTRY

Scientific registry for PCS domains, taxonomy, templates, future variables, observation metadata, provider metadata, and quality metadata.

## PCS_CONNECTORS

Documentation framework for external scientific data providers and future connector metadata.

## PCS_ENGINE

Core processing layer responsible for loading standardized observations, computing projections, constructing state outputs, and writing machine-readable artifacts.

## PCS_OBSERVATORY

User-facing display layer for PCS state, coverage, data freshness, source status, and future monitoring views.

## PCS_DATA

Benchmark and processed datasets used for reproducibility, validation, and operational demonstrations.

## PAPER

Manuscript, publication materials, tables, figures, references, and scientific writing artifacts.

## docs

Project documentation, architecture documents, procedures, roadmap material, and scientific design notes.

## assets

Static visual assets, icons, images, diagrams, and other non-code resources used by dashboards, documentation, or publications.
