# Add a Scientific Domain or Subdomain

This document describes how to propose a new scientific domain or subdomain.

## Scientific rationale

Explain why the proposed domain or subdomain is scientifically necessary for PCS.

## Boundary with existing domains

Describe how the proposal differs from existing domains and why it should not be represented as an existing subdomain.

## Relevant observation sources

List possible scientific data providers, observation systems, or datasets that may support the domain.

## PCS role

Explain how the domain contributes to the Planetary Common State, including whether it supports observation, projection, validation, uncertainty, or system-status interpretation.

## Review requirement

New domains and subdomains require scientific review before being added to the official registry taxonomy.

## Rules

- Do not add variables as part of a domain proposal.
- Do not implement connectors.
- Do not modify engine logic.
