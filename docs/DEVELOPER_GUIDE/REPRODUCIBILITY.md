# PCS Reproducibility Requirements

PCS reproducibility requires that data, transformations, code versions, and assumptions be documented clearly.

## Open data

Use open scientific data where possible. If restricted data are required, document access constraints.

## Versioned code

Code used for data processing, validation, or state construction should be versioned.

## Clear data provenance

Record provider, dataset name, URL or access path, acquisition date, version, and citation.

## No fabricated data

Missing values must not be invented, inferred, or silently filled.

## No hidden transformations

All filtering, aggregation, unit conversion, normalization, and clipping must be documented.

## Preserve raw observations

Raw observations should be preserved separately from processed or normalized outputs.

## Document normalization

Normalization rules should record reference values, critical values, units, clipping choices, and scientific rationale.

## Record update time

Every generated data product should record when it was created or updated.
