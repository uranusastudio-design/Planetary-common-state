# Add a PCS Connector

This document describes the future process for adding a new data connector. It does not implement any connector.

## Required Documentation

### Provider

Identify the scientific provider and its role in PCS.

### API endpoint

Document any available endpoint or data access path. Do not implement it in this guide.

### Authentication

Record whether access is open, token-based, account-based, or otherwise restricted.

### Update frequency

Describe the expected update cadence.

### Data format

Record expected file formats or service response formats.

### License

Document licensing and usage constraints.

### Citation

Record required provider or dataset citations.

### Failure handling

Describe how unavailable services, missing files, authentication failure, and malformed responses should be handled.

### Missing data policy

State how missing values should be preserved and reported. Missing data should not be fabricated.

## Rules

- No API is implemented here.
- No code is added here.
- No variables are added here.
