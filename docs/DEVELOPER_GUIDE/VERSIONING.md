# PCS Versioning Principles

PCS uses version labels to distinguish prototype maturity, registry maturity, connector maturity, engine integration, observatory maturity, and AI-assisted monitoring.

## v0.x Prototype

Early operational prototypes, demonstration workflows, initial datasets, and proof-of-concept interfaces.

## v1.x Registry and architecture

Scientific registry, taxonomy, templates, connector documentation, architecture documents, and developer guides.

## v2.x Data connectors

Documented and tested data connectors, acquisition workflows, provider metadata, and source-status handling.

## v3.x Engine integration

Integrated PCS Engine workflows using registered variables, connectors, projections, state construction, and reproducible outputs.

## v4.x Observatory and UI

User-facing dashboards, observatory panels, display rules, accessibility improvements, and status views.

## v5.x AI-assisted monitoring

AI-assisted monitoring modules, decision-support context, uncertainty-aware summaries, and validated interpretation tools.

## Rules

- Version numbers should not imply scientific validation unless validation artifacts exist.
- Prediction features should not be introduced before validated data and state workflows exist.
- Documentation and implementation versions should remain traceable.
