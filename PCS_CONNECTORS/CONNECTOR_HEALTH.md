# Connector Health

Connector Health describes future operational checks for scientific data access.

## Health Checks

| Check | Purpose |
|---|---|
| Availability | Confirms the provider endpoint, portal, or dataset is reachable. |
| Latency | Measures delay between observation time and connector availability. |
| Freshness | Confirms the latest available data are current relative to expected cadence. |
| Integrity | Confirms files, checksums, schema, and required fields are consistent. |
| Version | Confirms dataset or connector version is known and recorded. |

## Health States

Health checks should support clear reporting without implying scientific validation beyond connector operation.

Current framework status: documentation only.
