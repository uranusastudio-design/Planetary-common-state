# PCS Connector Schema

Every future PCS connector record should follow this schema.

| Field | Description |
|---|---|
| Provider | Name of the scientific data provider. |
| Website | Main provider website or data portal. |
| API Endpoint | Documented API or data endpoint, if available. |
| Authentication | Authentication requirements, if any. |
| Update Frequency | Expected cadence of data updates. |
| Variables | PCS variables or future variable groups supported by the provider. |
| Coverage | Spatial, temporal, or thematic coverage. |
| Spatial Resolution | Native or effective spatial resolution. |
| Temporal Resolution | Native or effective temporal resolution. |
| Latency | Delay between observation and availability. |
| Data Format | Common data formats made available by the provider. |
| License | Data license, usage terms, or access policy. |
| Citation | Required dataset or provider citation. |
| Status | Documentation, candidate, planned, implemented, deprecated, or unavailable. |
| Notes | Caveats, access constraints, quality notes, or integration considerations. |

No provider API is implemented in this schema.
