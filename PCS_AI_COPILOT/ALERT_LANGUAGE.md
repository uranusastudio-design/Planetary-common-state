# Alert Language

PCS alert language must be calm, specific, and review-oriented. The goal is to support monitoring without creating alarm fatigue.

## Alert Levels

### Normal

Use routine language. Indicate that available systems are operating as expected.

### Notice

Use informational language. Indicate a small change, expected delay, or low-priority issue.

### Attention

Use focused language. Indicate an item that should be reviewed when practical.

### Warning

Use clear operational language. Indicate a significant data-quality, connector, or monitoring issue that requires timely human review.

### Critical

Use concise, high-priority language. Indicate that an important system component or data stream requires immediate review.

## Language Principles

- Avoid sensational wording.
- Avoid unsupported hazard claims.
- Avoid implying causality without validation.
- State what is known, what is missing, and what should be reviewed.
- Keep alerts traceable to source status and confidence.
